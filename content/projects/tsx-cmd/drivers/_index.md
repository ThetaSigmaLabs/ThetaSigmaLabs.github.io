Each driver gets its own page with a status checklist, a running dev
log, and any backend-specific gotchas. The aggregate view lives on the
[tsx_cmd compatibility matrix](../#compatibility-matrix).

* [TheSkyX](theskyx/) — reference backend, JavaScript-over-TCP API.
* [INDIGO](indigo/) — cross-platform device framework.
* [ASCOM](ascom/) — Windows-side standard, Alpaca for network access.
* [Simulator](simulator/) — built-in, used as the conformance target.
* [Generic notes](generic/) — anything that applies across drivers
  (capability discovery, error mapping, units, threading model).
